# 02. Stakeholder & User Role Analysis

The Ruqi Store system recognizes and enforces four distinct user roles, each maintaining a strictly controlled scope of system access, business responsibilities, and operational boundaries.

| User Role | System Scope & Target Responsibilities |
| :--- | :--- |
| **Customer** | Registers an account, browses the dynamic product catalog, manages a personal shopping cart, securely places orders, tracks delivery pipelines, and leaves verified product reviews. |
| **Store Manager** | Handles day-to-day storefront operations, updates the product catalog, controls inventory levels, fulfills incoming orders, and actively manages the physical showroom appointment calendar. |
| **Accountant** | Focuses strictly on financial governance. Dedicated to reviewing payment submissions, cross-checking bank records, updating order billing statuses (Paid/Rejected), and maintaining immutable payment logs. |
| **Administrator** | Maintains comprehensive system oversight. Responsible for user management, administrative role assignment, moderating public product reviews, auditing security logs, and generating system-wide CSV reports. |
